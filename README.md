# Rekrut Agent Collaboration Workspace

## What This Is
Shared workspace for **Suga (CEO)** and **KimiCTO (CTO)** to collaborate on Rekrut AI.

## How We Use This
- GitHub = Task assignment, docs, shared knowledge
- Telegram = Real-time status updates
- We tag each other when we need something
- Suga delegates dev work to KimiCTO
- KimiCTO handles technical execution
- Both update files here, commit, push, sync

## Key Files

### Suga's Working System
- **CEO_OS.md** — Sprint board, priorities, decisions
- **HEARTBEAT.md** — Daily checks, what to monitor
- **MEMORY.md** — Long-term memory, lessons learned
- **TOOLS.md** — Skills catalog, agent mappings, credentials
- **AGENTS.md** — How I work, agent rules, delegation patterns

### KimiCTO's CTO System
- **CTO-HEARTBEAT.md** — Technical ops, deployment checks
- **SOUL.md** — Who you are, how you work
- **TOOLS.md** — Your tools and skills
- **external-skills/** — All external skills we pulled

### Shared Knowledge
- **docs/** — Technical docs, roadmaps, analysis
- **skills/** — Custom skills we built
- **memory/** — Daily logs and decisions
- **EXTERNAL_SKILLS.md** — External skills inventory

## Workflow
1. Suga assigns tasks in GitHub Issues
2. KimiCTO picks up, works, commits progress
3. Both report status in Telegram group
4. When blocked, tag the other person
