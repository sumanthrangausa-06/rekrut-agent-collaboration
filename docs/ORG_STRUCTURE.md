## Daily Agent Operations

### Morning Standup (8:00 AM UTC)
Every agent reports to their lead:
- What they completed yesterday
- What they're working on today
- Blockers

### Continuous Deployment (24/7)
- **CI/CD pipeline** runs on every push to `dev`
- **Auto-deploy** to staging after tests pass
- **Manual promote** to production after QA approval
- **Security scans** on every build

### Sprint Cycle (2 weeks)
- **Sprint Planning:** Monday week 1
- **Mid-sprint Review:** Friday week 1
- **Sprint Retro:** Friday week 2
- **Release:** Friday week 2

### Agent Rotation
- Every 2 weeks, agents rotate between tasks to prevent knowledge silos
- Pair programming: Senior agents (odd numbers) paired with Junior agents (even numbers)
- Code review: Every PR needs 2 agent approvals before merge

## How We Work (Suga's Rules)

### 1. No Code Monoliths
Any file over 1000 lines gets split. Period. `onboarding.js` at 3119 lines is already on the refactor list.

### 2. Security First
No feature ships if it has a P0 or P1 security finding. Zero tolerance for IDOR, XSS, CSRF, or auth bypass.

### 3. Test Before Merge
Every PR needs:
- Unit tests (coverage target: 80%)
- Integration tests
- Security scan (no new vulnerabilities)
- Performance check (no regression > 10%)
- QA signoff

### 4. Documentation is Code
Every feature ships with:
- API documentation (OpenAPI)
- Frontend component documentation (Storybook)
- User-facing documentation (help center)
- Admin documentation (runbook)

### 5. AI is Not Magic
Every AI feature has:
- Fallback path (what if AI is down?)
- Human-in-the-loop option (candidate can appeal, recruiter can override)
- Explainability (why did the AI say this?)
- Bias audit (did we discriminate?)
- Cost ceiling (max tokens per request, max cost per user)

### 6. 24/7 Operations
- Day shift (08:00-20:00 UTC): Human-supervised agent work
- Night shift (20:00-08:00 UTC): Automated tasks, testing, maintenance, monitoring
- Weekend: Security patches, dependency updates, long-running tasks
- Holidays: Reduced staff, critical-only monitoring

### 7. Communication Cadence
- **Daily:** Standup summaries (08:00 UTC)
- **Weekly:** Executive summary to CEO (Friday 17:00 UTC)
- **Bi-weekly:** Sprint review + retro
- **Monthly:** Board metrics report
- **Quarterly:** Strategic planning session
